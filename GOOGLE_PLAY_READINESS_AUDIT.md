# Google Play Readiness Audit

Audit date: June 18, 2026

Scope: Android readiness for the Baby Steps app codebase. This is an audit only. No Google Play submission, Play Console release, major feature implementation, or source-code refactor was performed.

## Executive Summary

Overall readiness rating: **Not ready**

Baby Steps has a credible Expo/React Native foundation and an EAS production profile that is configured to generate an Android App Bundle. TypeScript, Jest, and lint checks currently pass, although lint has many warnings. The app is not ready for Google Play upload because release identity, child-safety policy posture, privacy/data deletion readiness, final Android permission review, external child-facing WebView content, hardcoded third-party credentials, and release-build smoke testing are not complete.

Strict verdict:

- Ready: local validation commands, npm dependency baseline, Expo config presence, EAS production AAB profile, core docs, local privacy policy draft, account deletion draft page.
- Not ready: Play Console submission, production release, closed testing, and policy review.
- Risky: child/family target audience, child-facing YouTube/WebView, weak parent gate, media permissions, Supabase child profile/progress data, local `.env` handling, placeholder routes/content, final Android manifest not verified from a signed release AAB.
- Must fix before upload: remove/revoke hardcoded Sunbird credential, decide final app/package identity, host privacy/deletion URLs, add in-app account deletion path or request flow, gate/remove child-facing external links and share flows, inspect final AAB permissions, produce and install a release build.

## Manual QA Checklist For Google Play Internal Testing

Before uploading to a Play internal testing track:

- [ ] Build a production AAB with EAS.
- [ ] Confirm EAS credentials and Play App Signing setup.
- [ ] Inspect final AAB manifest for package name, version code, target SDK, min SDK, and permissions.
- [ ] Install via Play internal testing or bundletool-generated APKs on a physical Android device.
- [ ] Confirm first launch, splash, font loading, and Supabase session loading.
- [ ] Confirm signup, login, forgot password, reset password deep link, and logout.
- [ ] Provide Play Console reviewer credentials or app access instructions.
- [ ] Add a child profile and confirm Supabase row creation.
- [ ] Launch child mode and verify active child state survives expected navigation.
- [ ] Test Android hardware back in parent, child, story, game, museum, modal, and coloring screens.
- [ ] Verify child mode orientation and each game screen orientation.
- [ ] Complete each story page flow and quiz.
- [ ] Complete Word Game, Card Matching, Puzzle, Luganda Learning, and Luganda Counting smoke paths.
- [ ] Confirm activities/progress/achievements write and render as expected.
- [ ] Confirm achievement definitions are seeded or app handles missing definitions gracefully.
- [ ] Test audio playback, background/foreground transitions, and audio cleanup.
- [ ] Test coloring brush, eraser, fill, undo, redo, save, share, denied permission, and granted permission.
- [ ] Confirm media permission prompts are contextual and accurate.
- [ ] Remove or gate all child-facing YouTube/WebView video links, then retest museum screens.
- [ ] Test offline/no-network startup with previous session.
- [ ] Test poor-network behavior for Supabase-backed screens and activity writes.
- [ ] Confirm app restart behavior for local progress and active child state.

## Payment Readiness

Payments/subscriptions are not implemented. No payment SDKs, billing clients, subscription state, entitlements, or backend purchase verification were found. Store listing copy should not mention premium content, subscriptions, paywalls, or purchases as active features.

If Baby Steps later sells digital lessons, stories, games, packs, subscriptions, or unlocks in the Android app distributed through Google Play, the team should assume Google Play Billing requirements apply, with backend verification and entitlement storage. Do not implement payments client-only.

## Database And Content Readiness

Current state:

- Supabase supports auth, children, activities, achievements, and child-achievement joins.
- Curriculum/story/game/museum/lesson content is mostly bundled and hardcoded.
- `schema.sql` is explicitly a context snapshot, not an executable migration.
- Activities and progress do not have stable content/version IDs.
- MVP can temporarily ship with bundled local content only if content is complete, licensed, reviewed, and store claims are accurate.

Risks:

- Broken/missing hardcoded content requires app updates.
- Placeholder video URLs and sample progress can appear to users.
- Database schema/policies/migrations are not release-managed in repo.

## Final Verdict

Ready for Google Play internal testing: **No, not yet.** The project has an AAB-capable EAS production profile, but it needs P0 security, identity, child-safety, permission, account deletion, and release-build verification fixes before upload.

Ready for production: **No.** Production requires Play Console declarations, hosted legal/support URLs, child/family compliance review, final content review, QA evidence, and release operations readiness.
